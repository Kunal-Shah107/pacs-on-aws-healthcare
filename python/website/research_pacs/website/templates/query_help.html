A query is composed of one or more conditions, combined with AND or OR operands. You can also enclose multiple conditions and operands in parenthesis () in order to change the predecence of operands (e.g. (Condition1 OR Condition2) AND Condition3).

Each condition is defined as Tag Operator [Value] where a Value is required for some operators only.

Tag can be a case-sensitive keyword (e.g. Modality), a hexadecimal number (e.g. 7FE00010), or a sequence of tags separated by dots (e.g. SequenceOfUltrasoundRegions.RegionSpatialFormat, 00186011.RegionSpatialFormat)

Operator can be:

  • Exists (no value needed): The data element must exist
  • NotExists (no value needed): The data element must not exist
  • Empty (no value needed): At least one value of the data element must be empty
  • NotEmpty (no value needed): At least one value of the data element must not be empty
  • StrEquals (value is a string, wilcard * allowed): Must equal value, or match value if * are used (case insensitive)
  • StrNotEquals (value is a string, wilcard * allowed): Must not equal value, and must not match value if * are used (case insensitive)
  • NbEquals (value is a integer or decimal): Must equals value
  • NbNotEquals (value is a integer or decimal): Must not equal value
  • NbGreater (value is a integer or decimal): Must be greater than value
  • NbLess (value is a integer or decimal): Must be less than value

You can find further etails and examples of valid queries in the User Guide.
